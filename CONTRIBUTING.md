## Code of Conduct

This project adheres to an inclusive community. Please be respectful and considerate in all interactions.

## Getting Started

1. Fork the repository
2. Create a new branch for your contribution
3. Make your changes
4. Test your changes locally
5. Submit a pull request

## Content Guidelines

- Ensure all technical content is accurate and verified against authoritative sources
- Write for university students, researchers, and educators in AI and robotics
- Include practical examples and hands-on exercises
- All examples must be reproducible and verifiable
- Prefer peer-reviewed publications, official documentation, and standard textbooks

## Technical Requirements

- All code snippets must be tested and verified
- Code examples must be reproducible in simulation or on edge devices
- Follow Docusaurus markdown format with proper frontmatter and sidebar metadata
- Maintain Flesch-Kincaid grade level 10-12 for readability

## Style Guide

- Use APA citation style consistently
- Include diagrams, simulations, or images where relevant
- Structure content with clear learning objectives
- Include assessment materials for each module
- Write in an educational, accessible tone

## Pull Request Process

1. Ensure your changes address a specific issue or improvement
2. Update the documentation as needed
3. Add tests if applicable
4. Verify all existing tests still pass
5. Submit your pull request with a clear description

## Citation Standards

- Minimum 50% peer-reviewed or official sources
- Include at least 10 credible sources per chapter
- Properly format all citations in APA style
- Verify all sources are current and accessible

## Questions?

If you have questions about contributing, please open an issue in the repository.
