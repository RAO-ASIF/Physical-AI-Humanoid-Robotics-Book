# Glossary

## A

**Action** - A ROS 2 communication pattern for long-running tasks with feedback, consisting of a goal, feedback, and result.

**APA Style** - A documentation format used for academic citations, emphasizing author, date, and source information.

## B

**Behavior Tree** - A hierarchical structure used in robotics and AI for organizing and executing complex behaviors.

## C

**Citation** - A reference to a source used in academic or technical writing, following a specific format like APA style.

**Continuous Joint** - A URDF joint type that allows unlimited rotation around its axis.

## D

**DDS (Data Distribution Service)** - The middleware used by ROS 2 for communication between nodes.

**Digital Twin** - A virtual representation of a physical system used for simulation and analysis.

**Differential Drive** - A common wheeled robot configuration using independently controlled wheels on either side.

**Docker** - A containerization platform used for consistent development and deployment environments.

## E

**Embodied Intelligence** - The theory that intelligence emerges from the interaction between an agent and its environment.

## F

**Fixed Joint** - A URDF joint type that creates a rigid connection between two links with no movement.

## G

**Gazebo** - A physics simulation environment widely used in robotics research and development.

**GPT (Generative Pre-trained Transformer)** - A type of AI language model used for natural language processing tasks.

## H

**Humanoid Robot** - A robot designed with human-like characteristics, including bipedal locomotion and manipulator arms.

## I

**Isaac ROS** - NVIDIA's collection of hardware-accelerated perception and navigation packages for ROS 2.

**Isaac Sim** - NVIDIA's high-fidelity simulation environment for robotics.

**Inertial Properties** - Physical properties that define how an object responds to forces, including mass and inertia tensor.

## J

**Joint** - In URDF, a connection between two links that defines the kinematic and dynamic relationship between them.

## K

**Kinematics** - The study of motion without considering the forces that cause it.

## L

**LIDAR** - Light Detection and Ranging, a sensor technology that measures distances using laser light.

**Link** - In URDF, a rigid body with visual, collision, and inertial properties.

## M

**Middleware** - Software that provides common services and capabilities to applications beyond what's offered by the operating system.

## N

**Nav2** - The next-generation navigation stack for ROS 2, providing path planning and obstacle avoidance capabilities.

**Node** - A process that performs computation in a ROS system.

## P

**Perception Pipeline** - A series of processing steps that convert sensor data into meaningful information about the environment.

**Physical AI** - The integration of artificial intelligence with physical systems, particularly robotic platforms that interact with the real world.

**Prismatic Joint** - A URDF joint type that allows linear sliding motion along a specified axis.

## Q

**QoS (Quality of Service)** - Settings in ROS 2 that allow fine-tuning of communication behavior, including reliability and durability.

## R

**Revolute Joint** - A URDF joint type that provides rotational movement around a single axis with limited range.

**Robot Operating System (ROS)** - A flexible framework for writing robot software, providing services like hardware abstraction and message passing.

**ROS 2** - The second generation of the Robot Operating System, featuring improved architecture and middleware.

**rclpy** - The Python client library for ROS 2.

## S

**SDF (Simulation Description Format)** - The format used by Gazebo to define simulation worlds and objects.

**Sensor Fusion** - The process of combining data from multiple sensors to create a more accurate understanding of the environment.

**SLAM (Simultaneous Localization and Mapping)** - A technique that allows robots to build a map of an unknown environment while simultaneously tracking their location within it.

**Subsumption Architecture** - A robot control architecture that uses layered behaviors with higher layers able to suppress lower layers.

## T

**TF (Transform)** - The ROS system that keeps track of multiple coordinate frames over time.

**Topic** - A ROS communication pattern using a publish-subscribe model.

## U

**URDF (Unified Robot Description Format)** - An XML format for representing a robot model including links, joints, and properties.

**Unity** - A game development platform used for creating high-fidelity simulation environments for robotics.

## V

**VSLAM (Visual Simultaneous Localization and Mapping)** - SLAM using visual sensors like cameras.

**Vision-Language-Action (VLA)** - Systems that integrate visual perception, language understanding, and action execution.

## W

**Whisper** - OpenAI's automatic speech recognition system used for voice command processing.

## X

**Xacro** - XML Macros for URDF, allowing parameterization and macro expansion in robot descriptions.
